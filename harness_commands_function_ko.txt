AI 하네스 명령어 치트시트
예시 기능 이름: function

이 파일은 루트의 harness_commands_function.txt 한글판입니다.
아래 예시에서 function은 --feature 값입니다. 실제 작업에서는 기능을 구분하기 쉬운 영문 슬러그로 바꿔 쓰면 됩니다.


가장 자주 쓰는 명령
==================

빠른 모드로 새 작업을 끝까지 자동 실행:

python .ai\harness_fast.py run "<요청 내용>" --feature function --yes --defaults

풀 모드로 새 작업을 끝까지 자동 실행:

python .ai\harness.py run "<요청 내용>" --feature function --yes --defaults

멈춘 작업을 이어서 자동 진행:

python .ai\harness_fast.py auto function --yes --defaults
python .ai\harness.py auto function --yes --defaults

현재 작업 상태 확인:

python .ai\harness_fast.py status function
python .ai\harness.py status function

전체 작업 목록 확인:

python .ai\harness_fast.py list
python .ai\harness.py list


빠른 파이프라인
===============

빠른 모드는 세 단계로 진행됩니다.

00_specify -> 01_develop -> 02_verify

현재 권장 모델:

00_specify: Antigravity
01_develop: Claude
02_verify : Codex

빠른 모드 전체 자동 실행:

python .ai\harness_fast.py run "<요청 내용>" --feature function --yes --defaults

개발/검증 재시도 횟수를 더 크게 잡고 빠른 모드 실행:

python .ai\harness_fast.py run "<요청 내용>" --feature function --yes --defaults --max-verify-fix-retries 5

이미 만들어진 빠른 모드 작업을 이어서 실행:

python .ai\harness_fast.py auto function --yes --defaults

빠른 모드 공급자와 프리셋 상태 점검:

python .ai\harness_fast.py doctor


풀 파이프라인
=============

풀 모드는 일곱 단계로 진행됩니다.

00_specify -> 01_plan -> 02_develop -> 03_review -> 04_fix -> 05_verify -> 06_document

현재 권장 모델:

00_specify : Codex
01_plan    : Antigravity
02_develop : Claude
03_review  : Antigravity
04_fix     : Claude
05_verify  : Codex
06_document: Antigravity

기본 자동 실행:

python .ai\harness.py run "<요청 내용>" --feature function --yes

defaults 모드로 자동 실행:

python .ai\harness.py run "<요청 내용>" --feature function --yes --defaults

Verify/Fix 재시도 횟수를 더 크게 잡고 자동 실행:

python .ai\harness.py run "<요청 내용>" --feature function --yes --defaults --max-verify-fix-retries 5

타임아웃과 최대 단계 전환 수를 직접 지정해서 자동 실행:

python .ai\harness.py run "<요청 내용>" --feature function --yes --defaults --timeout 3600 --max-steps 30 --max-verify-fix-retries 3


수동 단계별 실행
================

1. 새 작업을 만들고 00_specify 프롬프트 생성:

python .ai\harness.py start "<요청 내용>" --feature function

2. 현재 프롬프트 파일 경로 확인:

python .ai\harness.py prompt function

3. 현재 프롬프트 내용을 터미널에 출력:

python .ai\harness.py prompt function --print

4. 출력된 프롬프트를 대상 모델에 직접 실행합니다.
   모델은 현재 단계에서 요구하는 산출물 파일을 반드시 작성해야 합니다.

5. 모델이 단계 산출물을 작성한 뒤 하네스 재개:

python .ai\harness.py resume function

6. 휴먼 게이트에서 막히면 승인:

python .ai\harness.py approve function

7. 완료될 때까지 아래 루프 반복:

python .ai\harness.py prompt function
python .ai\harness.py prompt function --print
# 프롬프트를 대상 모델에 직접 실행
python .ai\harness.py resume function

8. 언제든 현재 상태 확인:

python .ai\harness.py status function

9. 로그 확인 또는 실시간 추적:

python .ai\harness.py log function
python .ai\harness.py log function --follow

중간에 멈춘 작업 이어가기
=========================

이미 작업이 있고 중간에 멈춘 경우 현재 단계부터 자동 진행:

python .ai\harness.py auto function --yes

현재 단계부터 defaults 모드로 자동 진행:

python .ai\harness.py auto function --yes --defaults

Verify/Fix 재시도 제한을 더 크게 잡고 이어서 진행:

python .ai\harness.py auto function --yes --defaults --max-verify-fix-retries 5

휴먼 게이트에서 막힌 작업을 승인하고 자동 계속 진행:

python .ai\harness.py approve function --auto --yes

휴먼 게이트 승인 후 defaults 모드로 자동 계속 진행:

python .ai\harness.py approve function --auto --yes --defaults

모델 산출물은 이미 작성됐고 하네스만 아직 재개하지 않은 경우:

python .ai\harness.py resume function

모델 산출물을 반영한 뒤 바로 자동 계속 진행:

python .ai\harness.py resume function --auto --yes

Verify/Fix 재시도 제한에 도달해서 더 많은 반복을 허용하고 싶은 경우:

python .ai\harness.py resume function --auto --yes --max-verify-fix-retries 5


상태와 로그 확인
================

특정 작업 상태 자세히 보기:

python .ai\harness.py status function
python .ai\harness_fast.py status function

feature 인자를 생략하면 전체 작업 목록 보기:

python .ai\harness.py status
python .ai\harness_fast.py status

최근 로그 보기:

python .ai\harness.py log function
python .ai\harness_fast.py log function

최근 로그 줄 수 지정:

python .ai\harness.py log function --lines 200

로그를 계속 따라가기:

python .ai\harness.py log function --follow
python .ai\harness_fast.py log function --follow


유지보수 명령
=============

모든 실행 목록 보기:

python .ai\harness.py list
python .ai\harness_fast.py list

공급자와 프리셋 준비 상태 확인:

python .ai\harness.py doctor
python .ai\harness_fast.py doctor

실행 상태만 삭제하고 생성된 기능 파일은 보존:

python .ai\harness.py cleanup function --keep-feature
python .ai\harness_fast.py cleanup function --keep-feature

실행 상태와 생성된 기능 파일 모두 삭제:

python .ai\harness.py cleanup function
python .ai\harness_fast.py cleanup function


옵션 의미
=========

--feature function
작업 이름을 function으로 지정합니다. 실제로는 settings-tab, num-to-name 같은 영문 슬러그를 쓰면 됩니다.

--yes
자동 실행 중 휴먼 게이트가 나오면 가능한 경우 승인하고 계속 진행합니다.

--defaults
모델이 애매한 결정을 만났을 때 사용자에게 자주 멈춰 묻지 않고, 권장 기본값을 선택하게 합니다.

--timeout 3600
각 모델 실행의 최대 대기 시간을 초 단위로 지정합니다.

--max-steps 30
자동 실행 중 최대 단계 전환 횟수를 지정합니다.

--max-verify-fix-retries 5
검증 실패 후 수정 단계로 되돌아가는 반복 횟수를 지정합니다.

--print
현재 프롬프트 파일 경로만 보여주지 않고 프롬프트 본문까지 출력합니다.

--follow
로그를 한 번만 출력하지 않고 계속 따라갑니다.

--keep-feature
cleanup 때 .ai/runs/<feature>만 지우고 .ai/features/<feature>는 남깁니다.


파일 위치
=========

프롬프트:
.ai\runs\<feature>\prompts\

모델 stdout/stderr 로그:
.ai\runs\<feature>\logs\

하네스 이벤트 로그:
.ai\runs\<feature>\harness.log

단계 산출물:
.ai\features\<feature>\

문서 산출물:
.ai\docs\


새로 추가된 진단/재시도 명령
===========================

깊은 환경 점검:

python .ai\harness.py doctor --deep
python .ai\harness_fast.py doctor --deep

`doctor --deep`은 단순히 실행 파일이 있는지만 보지 않고, Codex/Claude/AGY에 짧은 테스트 프롬프트를 실제로 실행해 봅니다.
각 provider가 정상 응답하는지 확인하므로 일반 `doctor`보다 오래 걸릴 수 있습니다.

실행 중인 작업 감시:

python .ai\harness.py watch function
python .ai\harness_fast.py watch function

최근 로그 줄 수와 갱신 간격 지정:

python .ai\harness.py watch function --lines 12 --interval 3

완료 또는 blocked 상태가 되면 watch 종료:

python .ai\harness.py watch function --exit-on-stop

왜 멈췄는지 설명:

python .ai\harness.py explain function
python .ai\harness_fast.py explain function

현재 단계 재시도:

python .ai\harness.py retry function
python .ai\harness_fast.py retry function

기본 `retry`는 직전 실패 이유, 누락된 산출물, provider 로그, 검증 결과를 붙인 보강 프롬프트로 현재 단계를 다시 실행합니다.
재시도 전 기존 stage md/result.json은 `.ai\runs\<feature>\retry_archives\`에 보관하고 원래 위치는 비워서, 낡은 결과를 하네스가 다시 읽는 일을 막습니다.

기존 프롬프트 그대로 재시도:

python .ai\harness.py retry function --same

재시도 프롬프트만 만들고 provider는 실행하지 않기:

python .ai\harness.py retry function --prompt-only

재시도 성공 후 다음 단계들도 자동으로 계속 진행:

python .ai\harness.py retry function --auto --yes --defaults


Project Contract 후보 정산
===========================

standard/full 파이프라인은 완료 후 Project Contract 후보를 자동 추출합니다.

미정 상태의 PC 후보가 하나라도 있으면 fast/standard/full 모두 새 작업을 시작할 수 없습니다.

미정 후보를 Claude로 검토하고 승인/기각하려면:

python .ai\pc_review.py

이 명령은 인자를 받지 않습니다.
Claude가 미정 후보와 .ai\project_contract.md를 함께 읽고 의견을 제시합니다.
사용자가 Yes를 입력하면 Claude가 .ai\history\pc_candidates.json과 .ai\project_contract.md를 수정합니다.
사용자가 No를 입력하면 아무 파일도 바꾸지 않고 미정 후보가 그대로 남습니다.


단계 결과 JSON
==============

각 단계는 사람이 읽는 md 산출물과 하네스가 읽는 JSON 산출물을 같이 만들도록 바뀌었습니다.

예:

.ai\features\<feature>\00_spec.md
.ai\features\<feature>\00_spec.result.json

하네스는 `.result.json`을 먼저 읽고, 파일이 없으면 기존처럼 md의 `## 단계 결과` 블록을 읽습니다.
따라서 기존 산출물과 호환되지만, 앞으로는 md와 result.json을 둘 다 만드는 흐름이 권장됩니다.

result.json 기본 형태:

{
  "status": "PASS",
  "next_stage": "01_develop",
  "human_gate_required": false,
  "blocking_reason": ""
}


실패 인수인계 파일
==================

provider 실패, 산출물 누락, write policy 위반, verify 실패 같은 상황에서는 아래 파일이 자동 생성됩니다.

.ai\runs\<feature>\handoff.md

이 파일에는 실패 이유, 확인할 로그, 최근 이벤트, 다음 모델이 봐야 할 내용이 정리됩니다.
`retry`는 이 handoff 내용을 보강 프롬프트에 붙여서 같은 실수를 줄입니다.
