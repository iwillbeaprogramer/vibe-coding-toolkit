# 00_spec - initial-project

작성: Codex
일시: 2026-05-26

## 기능 목표
- Windows 데스크톱에서 종목 심볼을 검색하면 주식/ETF의 상세 시세, 기본 정보, 핵심 지표, 차트를 확인할 수 있는 초기 애플리케이션을 만든다.
- 프론트엔드는 WPF, 백엔드는 Python FastAPI로 분리하여 로컬에서 실행 가능한 구조를 갖춘다.

## 기능명
- feature_name: initial-project
- naming_reason: 하네스에서 기능명이 잠겨 있으며, 애플리케이션의 최초 프로젝트 생성과 기본 기능 정의를 포함하므로 기존 slug를 유지한다.

## 구체적 요구사항
- WPF 클라이언트는 사용자가 종목 심볼을 입력하고 검색할 수 있는 화면을 제공해야 한다.
- 기본 검색 입력은 `QLD` 같은 미국 주식/ETF 티커 문자열로 처리한다.
- 검색 입력은 앞뒤 공백을 제거하고 대문자로 정규화해서 백엔드에 전달한다.
- 비어 있는 검색어, 허용되지 않는 문자, 과도하게 긴 입력은 사용자에게 오류 메시지를 보여주고 요청을 보내지 않는다.
- FastAPI 백엔드는 WPF에서 호출할 HTTP API를 제공해야 한다.
- 백엔드는 종목 심볼 기준으로 상세 정보를 조회하고, WPF에서 바로 렌더링하기 쉬운 JSON 형태로 반환해야 한다.
- 초기 화면에는 최소한 현재가, 전일 종가, 시가, 고가, 저가, 거래량, 전일 대비 변동액, 변동률, 통화, 거래소, 조회 시각을 표시해야 한다.
- 가능한 경우 추가 지표로 52주 고가/저가, 시가총액, PER, EPS, 배당수익률, 베타, 평균 거래량, 종목명, 산업/섹터, 설명을 표시한다.
- 데이터 공급자에서 특정 필드를 제공하지 않으면 해당 항목은 `N/A` 또는 빈 상태로 표시하고 전체 검색 결과는 실패시키지 않는다.
- 차트는 가격 흐름을 확인할 수 있도록 최소 일봉 기준의 최근 기간 데이터를 표시해야 한다.
- 차트 기본 기간은 최근 6개월로 두고, 초기 구현에서 가능하면 1개월, 6개월, 1년 선택을 지원한다.
- 백엔드는 차트용 시계열 데이터를 날짜, 시가, 고가, 저가, 종가, 거래량 필드로 반환해야 한다.
- WPF는 백엔드 연결 실패, 데이터 공급자 실패, 종목 미존재, 요청 제한 가능성을 구분 가능한 메시지로 보여줘야 한다.
- 로컬 개발 환경에서 WPF 앱과 FastAPI 서버를 각각 실행할 수 있는 기본 구조와 실행 방법이 마련되어야 한다.
- 금융 데이터는 투자 자문이 아니라 정보 제공용임을 UI 또는 문서에서 명확히 해야 한다.

## 이번 범위에서 제외하는 것
- 실시간 스트리밍 시세: 초기 구현은 검색 시점의 조회 및 기간 차트 데이터로 제한한다.
- 사용자 계정, 로그인, 관심종목 저장, 포트폴리오 관리: 초기 검색/상세 조회 범위를 벗어난다.
- 매매 주문, 브로커 연동, 투자 추천/매수매도 신호: 금융 규제와 보안 위험이 크므로 제외한다.
- 유료 데이터 공급자 연동 및 API 키 관리: 기본 구현은 공개 또는 무료 사용 가능한 공급자 우선으로 계획하되, 제한 사항을 문서화한다.
- 배포 설치 프로그램, 자동 업데이트: 초기 프로젝트 구조와 기능 검증 이후 별도 단계로 다룬다.

## 입력과 출력
- 입력 형태: WPF 검색창의 문자열 티커 심볼.
- 입력 타입 및 범위: 영문 대소문자, 숫자, 점(`.`), 하이픈(`-`)을 포함하는 1~15자 문자열을 기본 허용한다.
- 입력 예시: `QLD`, `AAPL`, `MSFT`, `BRK-B`.
- 정상 출력: 종목 요약 정보, 상세 지표, 가격 정보, 차트 데이터가 포함된 JSON 응답과 이를 표시하는 WPF 화면.
- 백엔드 응답 기본 구조: `symbol`, `name`, `exchange`, `currency`, `quote`, `metrics`, `profile`, `chart`, `fetched_at`, `source`를 포함한다.
- 비정상 입력 처리: WPF에서 1차 검증 후 사용자 메시지를 표시하고, FastAPI도 400 응답으로 동일한 검증을 수행한다.
- 종목 미존재 처리: FastAPI는 404 또는 명확한 오류 코드와 메시지를 반환하고, WPF는 "종목을 찾을 수 없음" 상태를 표시한다.
- 외부 데이터 실패 처리: FastAPI는 502 또는 명확한 오류 코드와 메시지를 반환하고, WPF는 재시도 가능한 오류로 표시한다.

## 기존 코드 영향
- 현재 저장소에는 제품용 `src/` 또는 `tests/` 코드가 없고, 하네스/프리셋 파일만 존재한다.
- 다음 단계에서는 새 프로젝트 구조를 `src/` 하위에 만드는 계획이 필요하다.
- 예상 구조는 WPF 클라이언트와 FastAPI 백엔드를 분리하되, 저장소 내부에서 함께 관리하는 방식이다.
- 기존 하네스 파일, 프리셋, 실행 상태 파일은 기능 구현 대상이 아니며 수정하지 않는다.
- 충돌 가능성이 있는 기존 제품 기능은 확인되지 않았다.

## 기술적 제약
- 프론트엔드는 WPF 기반 Windows 데스크톱 애플리케이션으로 구현한다.
- 백엔드는 Python FastAPI로 구현하고 JSON HTTP API를 제공한다.
- WPF와 FastAPI는 로컬 개발 환경에서 서로 통신할 수 있어야 한다.
- 차트 UI는 WPF에서 안정적으로 렌더링 가능한 검증된 차트 라이브러리 사용을 우선 검토한다.
- 주식 데이터 조회는 무료/공개 데이터 소스를 기본 후보로 검토하되, 사용량 제한과 지연 시세 가능성을 문서화한다.
- 네트워크 장애와 외부 API 제한에 대비해 백엔드에서 명확한 예외 변환을 해야 한다.
- 금융 데이터의 정확성, 지연 여부, 데이터 출처를 사용자에게 표시해야 한다.
- 새 외부 의존성이 필요할 가능성이 높으므로 다음 계획 단계에서 패키지와 라이선스/제약을 확정해야 한다.
