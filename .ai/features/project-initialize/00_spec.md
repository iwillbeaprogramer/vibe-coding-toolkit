# 00_spec - project-initialize

작성: Codex
일시: 2026-05-27

## 기능 목표
- WPF 데스크톱 앱에서 사용자가 주식/ETF 티커를 검색하면 Python FastAPI 백엔드가 시세, 기본 지표, 기업/상품 개요, 차트 데이터를 제공하고 WPF가 이를 상세 화면으로 표시한다.
- 초기 버전은 Windows 로컬 실행을 기준으로 하며, QLD 같은 미국 상장 티커를 우선 지원한다.

## 기능명
- feature_name: project-initialize
- naming_reason: 하네스에서 기능명이 잠겨 있으므로 제공된 slug를 그대로 사용한다.

## 구체적 요구사항
- WPF 앱은 검색 입력창, 검색 실행 버튼, 로딩/오류 상태, 상세 정보 영역, 차트 영역을 제공해야 한다.
- 사용자는 티커 문자열을 입력해 검색할 수 있어야 한다. 입력값은 앞뒤 공백을 제거하고 대문자 정규화 후 API에 전달한다.
- 기본 지원 범위는 미국 상장 주식과 ETF 티커이며, 예시 입력 `QLD`가 정상 조회되어야 한다.
- FastAPI 백엔드는 WPF에서 호출할 수 있는 REST API를 제공해야 한다.
- API는 단일 티커 조회 엔드포인트를 제공해야 한다. 기본안은 `GET /api/stocks/{symbol}`이다.
- API 응답에는 현재가, 전일 종가, 시가, 고가, 저가, 거래량, 평균 거래량, 시가총액, 52주 고가/저가, PER, EPS, 배당수익률, 거래소, 통화, 종목명, 섹터/산업 또는 ETF 유형, 설명, 데이터 기준 시각을 포함한다.
- API 응답에는 차트 렌더링에 필요한 OHLCV 배열을 포함해야 한다. 기본 기간은 6개월, 기본 간격은 1일 봉으로 한다.
- WPF 상세 화면은 주요 가격 정보를 상단 요약으로 보여주고, 세부 지표는 표 또는 그리드 형태로 보여주며, 차트는 시간순 가격 흐름을 확인할 수 있게 표시해야 한다.
- 데이터 조회 실패, 티커 미존재, 네트워크 오류, API 서버 미실행 상태는 WPF에서 사용자가 이해할 수 있는 오류 메시지로 표시해야 한다.
- 백엔드는 외부 데이터 공급자가 일시적으로 실패하는 경우 명확한 HTTP 오류와 구조화된 오류 응답을 반환해야 한다.
- 초기 구현은 투자 조언이 아니라 정보 조회 도구임을 UI 또는 앱 메타 정보에서 명확히 해야 한다.
- WPF와 FastAPI는 로컬 개발 환경에서 각각 실행 가능해야 하며, 연결 주소는 설정 가능해야 한다.
- 구현 후에는 WPF 빌드, FastAPI 기본 동작, API 응답 모델, 비정상 입력 처리에 대한 검증이 가능해야 한다.

## 이번 범위에서 제외하는 것
- 실시간 WebSocket 스트리밍 시세: 초기 버전은 검색 시점 조회와 필요 시 수동 새로고침으로 제한한다.
- 사용자 계정, 관심종목 저장, 포트폴리오 관리, 매매 기능: 정보 조회 앱의 초기 범위를 넘는다.
- 유료 시세 API 연동 및 API 키 기반 배포 설정: 자격 증명이 필요할 수 있어 초기 기본안에서는 무료/공개 데이터 소스를 우선한다.
- 한국/미국 외 모든 거래소의 완전 지원: 티커 형식과 데이터 품질 차이가 커서 미국 상장 종목/ETF를 먼저 안정화한다.
- 설치 패키지, 자동 업데이트, 운영 배포: 초기 개발 및 로컬 실행 검증 이후 별도 기능으로 분리한다.
- 금융 자문, 추천, 매수/매도 신호 생성: 사용자 요청 범위가 정보 조회와 차트 표시이므로 제외한다.

## 입력과 출력
- WPF 입력:
  - `symbol`: 문자열, 필수, 기본 허용 길이 1~15자.
  - 허용 문자는 영문, 숫자, 점, 하이픈을 기본으로 한다.
  - 입력 전처리는 trim 및 uppercase 정규화를 적용한다.
- API 입력:
  - `GET /api/stocks/{symbol}`
  - 선택 쿼리 기본안: `range=6mo`, `interval=1d`
- API 정상 출력:
  - `symbol`, `name`, `asset_type`, `exchange`, `currency`
  - `quote`: 현재가, 전일 종가, 가격 변동, 등락률, 시가, 고가, 저가, 거래량, 기준 시각
  - `fundamentals`: 시가총액, PER, EPS, 배당수익률, 52주 고가/저가, 평균 거래량 등 사용 가능한 지표
  - `profile`: 섹터, 산업, 설명 또는 ETF 설명
  - `chart`: OHLCV 포인트 배열
- API 오류 출력:
  - 400: 입력 형식이 잘못된 경우
  - 404: 조회 가능한 종목을 찾지 못한 경우
  - 502 또는 503: 외부 데이터 공급자 실패 또는 일시적 장애
  - 오류 응답은 `code`, `message`, `details` 필드를 가진 JSON으로 반환한다.
- WPF 출력:
  - 검색 결과 상세 화면, 차트, 로딩 상태, 오류 상태.
  - API 응답 필드가 없거나 외부 데이터 소스에서 제공되지 않는 값은 `N/A`로 표시한다.

## 기존 코드 영향
- 현재 `src/stocks-dashboard/stocks-dashboard.sln`과 `src/stocks-dashboard/stocks-dashboard/`에 .NET 8 WPF 프로젝트가 존재한다.
- WPF의 현재 `MainWindow.xaml`은 빈 `Grid` 상태이므로 검색 UI와 상세/차트 UI를 이 파일 중심으로 확장할 가능성이 높다.
- 현재 `MainWindow.xaml.cs`는 초기 생성자만 있으므로 검색 이벤트, API 호출, 상태 갱신 로직이 추가될 가능성이 높다.
- WPF 프로젝트 파일 `stocks-dashboard.csproj`에는 외부 패키지가 없으므로 HTTP 클라이언트 또는 차트 라이브러리 사용 여부를 다음 단계에서 결정해야 한다.
- FastAPI 백엔드는 아직 없으므로 루트 `src/` 하위에 새 Python 백엔드 구조를 추가하는 계획이 필요하다.
- 테스트 디렉터리 `tests/`는 현재 보이지 않으므로 다음 단계에서 백엔드 테스트와 필요한 경우 WPF 테스트/검증 방식을 계획한다.
- `.ai/project_contract.md`는 이미 변경된 상태로 보이며, 이 단계에서는 해당 파일을 수정하지 않는다.

## 기술적 제약
- 프론트엔드는 기존 프로젝트에 맞춰 WPF, .NET 8 Windows 타깃을 유지한다.
- 백엔드는 Python FastAPI를 사용한다.
- 외부 시세 데이터는 초기 기본안으로 API 키가 필요 없는 데이터 공급자를 우선 사용한다. 예: `yfinance`. 단, 구현 단계에서 안정성, 라이선스, 지연 시세 여부를 명시한다.
- 차트는 WPF 안에서 표시되어야 한다. 기본안은 WPF용 검증된 차트 라이브러리 사용을 검토하되, 새 외부 의존성은 계획 단계에서 명확히 기록한다.
- FastAPI 서버와 WPF 앱은 로컬 개발 기준으로 동작한다. 기본 API 주소는 `http://127.0.0.1:8000`으로 둔다.
- 외부 입력, 파일, 네트워크, 프로세스 실행 결과는 실패 가능성을 명시적으로 처리해야 한다.
- 프로덕션 코드는 루트 `src/` 하위, 테스트 코드는 루트 `tests/` 하위에만 둔다.
- Git commit, amend, reset, checkout, rebase, push는 실행하지 않는다.

## 사용자 확인 사항
- defaults_mode가 true이므로 사용자 질문으로 멈추지 않고 아래 기본 결정을 적용했다.
- 기본 결정: 기능명은 하네스가 잠근 `project-initialize`를 유지한다.
- 기본 결정: 초기 지원 시장은 미국 상장 주식/ETF로 제한한다. 이유: 사용자가 예시로 든 `QLD`가 미국 ETF이고, 초기 데이터 소스와 티커 규칙을 단순화할 수 있다.
- 기본 결정: 초기 데이터 조회는 실시간 스트리밍이 아니라 검색 시점 조회와 차트 데이터 조회로 한다. 이유: 구현 범위를 줄이고 무료 데이터 소스의 제약을 피한다.
- 기본 결정: 기본 차트 범위는 6개월, 간격은 1일 봉으로 한다. 이유: 초기 상세 화면에서 가격 흐름을 충분히 보여주면서 응답 크기를 관리할 수 있다.
- 기본 결정: 유료 API 키가 필요한 공급자는 제외하고 무료/공개 데이터 공급자를 우선한다. 이유: 현재 자격 증명이 제공되지 않았다.
- 기본 결정: 금융 추천/매매 신호는 제외한다. 이유: 사용자 요청은 주식 정보 검색과 상세 조회, 차트 표시다.
