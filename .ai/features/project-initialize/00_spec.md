# 00_spec - project-initialize

작성: Codex
일시: 2026-05-28

## 기능 목표
- 사용자가 Windows 환경에서 종목 티커 또는 종목명을 검색해 주식/ETF의 핵심 투자 정보를 확인할 수 있는 로컬 애플리케이션을 만든다.
- 프론트엔드는 React, 백엔드는 Python FastAPI로 구성하며, 검색 결과, 상세 지표, 가격 차트, 오류 상태를 한 화면 흐름으로 제공한다.

## 기능명
- feature_name: project-initialize
- naming_reason: 하네스에서 feature_name이 잠겨 있으므로 제공된 `project-initialize`를 그대로 사용한다.

## 구체적 요구사항
- React 기반 화면을 제공한다.
- 사용자는 검색 입력창에 `QLD` 같은 티커 또는 종목명 일부를 입력할 수 있어야 한다.
- 검색어가 비어 있거나 너무 짧으면 요청을 보내지 않고 사용자에게 입력 오류를 표시한다.
- 검색 결과는 티커, 종목명, 거래소, 자산 유형(주식/ETF 등)을 구분해서 보여준다.
- 사용자가 검색 결과 또는 직접 입력 티커를 선택하면 FastAPI 백엔드에서 상세 정보를 조회한다.
- 상세 화면에는 최소한 현재가, 전일 종가, 시가, 고가, 저가, 거래량, 평균 거래량, 시가총액, 52주 고가/저가, 통화, 거래소, 섹터/산업(가능한 경우), 배당 관련 값(가능한 경우), PER/EPS 등 대표 밸류에이션 값(가능한 경우)을 보여준다.
- ETF인 경우 운용보수, 순자산, 레버리지/상품 설명 등 데이터 공급원이 제공하는 ETF 관련 값이 있으면 함께 보여준다.
- 데이터 공급원이 제공하지 않는 값은 화면에서 누락처럼 보이지 않도록 `데이터 없음` 상태로 표시한다.
- 상세 화면에는 가격 차트를 제공한다.
- 차트는 기간 선택을 지원한다. 기본 기간은 1개월이며, 최소 선택지는 1일, 1개월, 6개월, 1년, 5년이다.
- 차트는 종가 기반 라인 차트를 기본으로 하고, OHLC 데이터가 있으면 툴팁에서 시가/고가/저가/종가/거래량을 확인할 수 있게 한다.
- 백엔드는 외부 데이터 조회 실패, 알 수 없는 티커, 제한 초과, 네트워크 오류를 구분 가능한 HTTP 오류로 반환한다.
- 프론트엔드는 로딩, 빈 결과, 오류, 재시도 상태를 제공한다.
- 앱은 투자 조언이 아니라 정보 조회용이라는 짧은 안내를 화면 하단 또는 상세 정보 주변에 표시한다.
- Windows 사용자가 개발 모드에서 실행할 수 있도록 백엔드와 프론트엔드 실행 절차가 명확해야 한다.
- 테스트는 백엔드 API의 정상/오류 응답과 프론트엔드 주요 렌더링 상태를 중심으로 추가한다.

## 이번 범위에서 제외하는 것
- 실시간 호가 스트리밍: 무료/무인증 데이터 공급원에서는 안정적인 실시간성이 보장되지 않으므로 초기 버전에서는 지연 시세 또는 공급원 제공 시세를 사용한다.
- 매수/매도 주문, 포트폴리오 관리, 알림, 로그인: 주식 정보 조회 앱의 초기 범위를 넘어서며 인증/보안 설계가 필요하다.
- 유료 API 키 기반 데이터 연동: 사용자에게 별도 자격 증명을 요구하지 않는 기본 구현을 우선한다.
- 정식 설치 파일 생성, 코드 서명, 자동 업데이트: Windows 프로그램 요구는 로컬 실행 가능한 앱으로 먼저 충족하고, 패키징은 후속 기능으로 분리한다.
- 금융 데이터의 완전성 보증: 외부 공급원 데이터의 누락/지연 가능성을 UI와 오류 처리로 명시한다.

## 입력과 출력
- 입력 형태: 검색창 문자열. 기본 허용 범위는 1~20자이며, 영문 티커, 숫자, 점, 하이픈, 공백을 허용한다.
- 입력 예시: `QLD`, `AAPL`, `Tesla`, `BRK-B`.
- 백엔드 검색 API 출력: 검색 후보 배열. 각 항목은 `symbol`, `name`, `exchange`, `assetType`, `currency`를 가능한 범위에서 포함한다.
- 백엔드 상세 API 출력: 종목 기본정보, 최신 가격 요약, 주요 재무/거래 지표, 차트 데이터 배열, 데이터 기준 시각, 데이터 공급원 정보를 포함한 JSON.
- 차트 데이터 출력: 시간값, 시가, 고가, 저가, 종가, 거래량을 포함한 배열. 일부 값이 없으면 null을 허용한다.
- 비정상 입력 처리: 빈 검색어는 400 또는 프론트엔드 검증 오류, 존재하지 않는 종목은 404, 외부 공급원 오류는 502, 일시적 제한/타임아웃은 503 계열로 처리한다.

## 기존 코드 영향
- 현재 `src/`와 `tests/`에는 확인 가능한 제품 코드와 테스트 코드가 없다.
- 다음 단계에서 제품 코드는 `src/` 하위에만 생성한다.
- 다음 단계에서 테스트 코드는 `tests/` 하위에만 생성한다.
- 하네스 파일, 프리셋, `.ai/runs/`, `.ai/history/`는 기능 구현 대상이 아니며 수정하지 않는다.
- 기존 기능과의 충돌 가능성은 낮지만, 새 React/FastAPI 프로젝트 구조와 의존성 파일을 추가해야 하므로 초기 프로젝트 구조 설계가 필요하다.

## 기술적 제약
- 프론트엔드 기본값: React + TypeScript + Vite.
- 백엔드 기본값: Python FastAPI.
- 차트 기본값: React에서 사용할 수 있는 검증된 차트 라이브러리. 다음 계획 단계에서 의존성 규모와 유지보수성을 비교해 선택한다.
- 금융 데이터 기본값: API 키 없이 개발 가능한 데이터 공급원 또는 라이브러리를 우선 검토한다. 데이터 지연/누락 가능성을 명시적으로 처리한다.
- 모든 외부 입력, 네트워크 호출, 프로세스 실행 결과는 실패 가능성을 명시적으로 처리한다.
- Windows 로컬 개발 환경을 우선 지원한다.
- 새 코드는 기존 Project Contract에 따라 `src/`와 `tests/` 경계를 지킨다.
- 루트 수준 제품 파일은 기존 프로젝트 관례 또는 실행에 필요한 구성 파일에 한정한다.

## 기본 결정 기록
- 사용자의 "윈도우 프로그램" 요구는 초기 범위에서 Windows에서 실행 가능한 로컬 React/FastAPI 앱으로 해석한다.
- 정식 데스크톱 패키징은 새 외부 의존성과 배포 정책 결정이 커서 이번 초기 구현 범위에서는 제외한다.
- API 키가 필요한 공급원은 기본값에서 제외한다. 사용자 자격 증명 없이 실행 가능한 구현을 우선한다.
- 상세 지표는 데이터 공급원이 제공하는 범위에서 최대한 표시하고, 누락 값은 오류가 아닌 `데이터 없음`으로 표시한다.
- `feature_name_locked: true`이므로 기능명은 `project-initialize`로 유지한다.
