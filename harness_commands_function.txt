AI Harness command cheat sheet
Feature name: function

Fast pipeline
=============

Fast mode uses three stages:

00_specify -> 01_develop -> 02_verify

Recommended models:

00_specify: Gemini
01_develop: Claude
02_verify : Codex

Run all fast stages automatically:

python .ai\harness_fast.py run "요청 내용을 여기에 작성" --feature function --yes --defaults

Run fast mode with a longer develop/verify retry loop:

python .ai\harness_fast.py run "요청 내용을 여기에 작성" --feature function --yes --defaults --max-verify-fix-retries 5

Continue an existing fast run:

python .ai\harness_fast.py auto function --yes --defaults

Check fast provider/preset setup:

python .ai\harness_fast.py doctor


Full pipeline
=============

Run all stages automatically
----------------------------

Default automatic run:

python .ai\harness.py run "요청 내용을 여기에 작성" --feature function --yes

Automatic run with defaults mode:

python .ai\harness.py run "요청 내용을 여기에 작성" --feature function --yes --defaults

Automatic run with a longer Verify/Fix retry loop:

python .ai\harness.py run "요청 내용을 여기에 작성" --feature function --yes --defaults --max-verify-fix-retries 5

Automatic run with custom timeout and max stage transitions:

python .ai\harness.py run "요청 내용을 여기에 작성" --feature function --yes --defaults --timeout 3600 --max-steps 30 --max-verify-fix-retries 3


Manual stage-by-stage execution
===============================

1. Create the run and generate the 00_specify prompt:

python .ai\harness.py start "요청 내용을 여기에 작성" --feature function

2. Check the current prompt path:

python .ai\harness.py prompt function

3. Optional: print the current prompt content:

python .ai\harness.py prompt function --print

4. Run the shown prompt with the target model manually.
   The model must write the required output file for the current stage.

5. Resume after the model writes the stage output:

python .ai\harness.py resume function

6. If the harness blocks on a human gate, approve it:

python .ai\harness.py approve function

7. Repeat this loop until complete:

python .ai\harness.py prompt function
python .ai\harness.py prompt function --print
# run the prompt manually with the target model
python .ai\harness.py resume function

8. Check current status at any time:

python .ai\harness.py status function

9. Watch or inspect logs:

python .ai\harness.py log function
python .ai\harness.py log function --follow

Continue automatically from the current stopping point
=====================================================

If the run already exists and stopped in the middle, continue from the current stage:

python .ai\harness.py auto function --yes

Continue from the current stage with defaults mode:

python .ai\harness.py auto function --yes --defaults

Continue from the current stage with a larger Verify/Fix retry limit:

python .ai\harness.py auto function --yes --defaults --max-verify-fix-retries 5

If the run is blocked by a human gate and you want to approve and continue automatically:

python .ai\harness.py approve function --auto --yes

If the run is blocked by a human gate and you want to approve and continue with defaults mode:

python .ai\harness.py approve function --auto --yes --defaults

If a model output was already written but the harness has not resumed yet:

python .ai\harness.py resume function

If a model output was already written and you want to resume, then continue automatically:

python .ai\harness.py resume function --auto --yes

If Verify/Fix retry limit was reached and you want to allow more iterations:

python .ai\harness.py resume function --auto --yes --max-verify-fix-retries 5


Useful maintenance commands
===========================

List all runs:

python .ai\harness.py list

Show provider availability:

python .ai\harness.py doctor

Remove only the run state and keep generated feature files:

python .ai\harness.py cleanup function --keep-feature

Remove the run state and generated feature files:

python .ai\harness.py cleanup function


Project Contract candidate review
=================================

Standard and full pipelines extract Project Contract candidates after completion.

If any candidate is still pending (`미정`), all new fast/standard/full runs are blocked.

Review pending candidates with Claude:

python .ai\pc_review.py

This script takes no arguments. Claude reads .ai\history\pc_candidates.json and .ai\project_contract.md, proposes approve/reject/ambiguous decisions, and asks for a Yes/No response.

If you answer Yes, Claude updates .ai\history\pc_candidates.json and .ai\project_contract.md.
If you answer No, no files are changed and pending candidates remain.
